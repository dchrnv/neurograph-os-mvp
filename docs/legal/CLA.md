## Definitions

### 1. "Contribution"

Any original work of authorship, including any modifications or additions to existing work, that is submitted by You to the Project Owner for inclusion in the NeuroGraph project. This includes, but is not limited to:

#### A. Source Code
- Rust, Python, JavaScript, or any programming language code
- Libraries, modules, packages, and dependencies
- Build scripts, CI/CD configurations, and automation tools
- Shell scripts, deployment configurations, and infrastructure code

#### B. Machine Learning Assets
- **Model Weights**: Pre-trained neural network weights, embeddings, parameters
- **Model Architectures**: Network designs, layer configurations, topologies
- **Training Datasets**: Labeled or unlabeled data used for training models
- **Fine-tuning Data**: Datasets for transfer learning or domain adaptation
- **Evaluation Datasets**: Benchmarks, test sets, validation data
- **Model Configurations**: Hyperparameters, training recipes, optimization settings

#### C. Data Assets
- **CDNA (Constitutional DNA)**: Behavioral rules, constraints, value systems
- **Semantic Graphs**: Knowledge graphs, ontologies, taxonomies
- **Token Structures**: Semantic tokens, embeddings, vector representations
- **Pre-configured Systems**: Pre-built NeuroGraph instances with trained structures

#### D. Documentation & Content
- Technical specifications, API documentation, tutorials
- Code comments, docstrings, inline documentation
- Examples, sample code, demonstration projects
- User guides, developer guides, deployment guides
- Architecture diagrams, design documents, RFCs

#### E. Test Assets
- Unit tests, integration tests, end-to-end tests
- Benchmarking suites, performance tests, stress tests
- Test fixtures, mock data, test utilities

#### F. Configuration & Deployment
- Docker configurations, Kubernetes manifests
- Monitoring configurations (Prometheus, Grafana, Jaeger)
- Environment configurations, deployment scripts
